title: Instant hero entrances (Webflow)
description: Make above-the-fold hero animations paint on the first frame as pure CSS — before GSAP loads — with aa-trigger="instant". Element-level fade/slide/blur/zoom plus an optional no-GSAP splitter for per-character text cascades.
---

A normal `aa-trigger="load"` entrance is GSAP-driven, so its first frame can't paint until the GSAP bundle (and, for split text, the webfont) has downloaded and `init()` has run — often a second or two on a cold load. The hero sits hidden behind the FOUC guard the whole time.

`aa-trigger="instant"` removes that wait. The entrance is a **CSS keyframe** inlined in your `<head>`, so it paints on the **first frame the browser renders** — before GSAP exists. The keyframe runs to completion on its own (the bundle arriving in 100 ms or 3 s makes no difference); when the library loads it builds *no* GSAP tween for these elements, so there's no re-animation or flash.

Use it for above-the-fold heroes. For cinematic line-text or anything below the fold, stay on `load` (full GSAP) — see [when not to use it](#when-not-to-use-it-use-load-instead).

## What you can animate

- **Element-level** (no extra script): `fade`, `fade-up` / `-down` / `-left` / `-right`, `zoom-in` / `zoom-out`, `blur`, `slide-up` / `-down` / `-left` / `-right`, `rotate*`. The motion comes from `aa-animate`; the eases match the GSAP named eases.
- **Per-character text** (with the loader, below): `text-fade`, `text-fade-up`/`-down`/`-left`/`-right`, `text-blur`, `text-scale` — split into characters and staggered, all in CSS.
- **Line-based text** (`text-slide-*`, `text-tilt-*`, `text-oval-*`, `text-rotate`) can **not** be instant — line breaks depend on layout and the final webfont, which aren't known before the bundle loads. Under `instant` they gracefully fall back to an element-level fade; use plain `load` (GSAP) if you need the real line effect.

```html
<h1 aa-animate="fade-up" aa-trigger="instant" aa-delay="0.1">
  Instant headline
</h1>
```

## 1. Head snippet (paste into Project Settings → Custom Code → Head Code)

This is the **complete** head block for an instant-hero site — FOUC guard, the `aa-timeout` safety net, the instant entrance CSS, the loader, and the `<noscript>` reveal. It's leaner than the [slow-network fallback](/recipes/load-fallback/) snippet because it **drops the whole `aa-fallback` staircase** (the 1s keyframe + polling timer): `instant` heroes self-reveal via CSS, so the only safety the rest of the page needs is the `aa-timeout` net in case JS never runs.

Everything the *first frame* depends on (keyframe, eases, split-unit layout) is inlined, since anything that only arrives in the non-blocking `dist` stylesheet isn't present at first paint. Paste it **above** the alrdy-animate `<script>` / `<link>` tags.

> **Only using `instant`?** This is all you need — no `aa-fallback`, and no separate `lcp` (the keyframe's `0.01` opacity floor already makes instant heroes LCP-eligible). Add the [GSAP-load fallback](/recipes/load-fallback/) block *only* if you also use `load` / `load-once` heroes (merge them — one shared FOUC guard).

## Combining with the slow-network fallback

If a site also uses GSAP `load` / `load-once` heroes with the [slow-network fallback](/recipes/load-fallback/), **merge both into one `<head>` block** — keep a single FOUC guard (the one above, excluding `lcp` + `instant`) and concatenate the rest. No conflict: the two systems act on disjoint triggers — the fallback timer targets `load` / `load-once`, the loader + instant CSS target `instant`, and instant elements self-reveal (the `aa-timeout` blanket reveal is a harmless no-op for them).

:::tip[Prefer a CDN tag over pasting the blob?]
Swap the inline `<script>` for `<script src="https://cdn.jsdelivr.net/npm/alrdy-animate@{{version}}/dist/loader.iife.js"></script>`. Simpler to keep updated, but it's one network request — so the first char-split paints a beat later than the inlined version. Element-level `instant` (and the `aa-delay` staircase) is pure CSS plus tiny inline JS, so the choice only affects the char cascade.
:::

## Per-element timing — `aa-delay`, `aa-duration`, `aa-ease`

The loader reads these three attributes and maps them to the custom properties the keyframe uses, so they work just like on a GSAP entrance:

- **`aa-delay="<s>"`** → `--aa-load-delay` — stagger a hero (headline, then subhead, then CTA). Any value; on split text it adds on top of the per-char stagger.
- **`aa-duration="<s>"`** → `--aa-load-dur` — entrance length (default `0.6s`).
- **`aa-ease="<name>"`** → `--aa-load-ease` — one of the CSS-expressible named eases (`osmo`, `energy`, `smooth`, `punch`, `relaxed`, `jump`, `pop`, `anticipate`, `fade`) or a raw CSS timing function. GSAP-only eases (`elastic`, `bounce`, `power*`) can't be expressed in CSS and fall back to the default.

These need the loader. Without it, set the props by hand: `style="--aa-load-delay:.1s; --aa-load-dur:.8s; --aa-load-ease:var(--aa-ease-smooth)"`.

## Fonts

Making the hero instant *exposes* a font swap that the old slow reveal used to hide: with `font-display: swap` the text paints in the fallback font, then swaps to the webfont a beat later. For **char-split** text this also re-wraps the line when the font swaps. **Preload the hero webfont** (the `<link rel="preload">` above) so it's available on the first frame — this is required for `instant` on text and worth it everywhere.

## Customising the motion

Every knob is a CSS custom property, and they're all declared in the `:root` block at the top of the snippet — edit them there to retune globally (no values are hidden in the loader JS; it only splits text + maps the attributes). To go finer:

- **Globally** — change `--aa-load-dur`, `--aa-load-ease`, `--aa-load-cstagger`, `--aa-load-opacity` etc. in `:root`.
- **Per direction** — the `[aa-animate~="…"]` / `[aa-animate^="text-"]` rules override `--aa-load-x` / `-y` / `-scale` / `-blur` per `aa-animate` value (e.g. how far a `fade-up` rises, the char-cascade distance).
- **Per element** — the `aa-duration` / `aa-ease` / `aa-delay` attributes, or any custom property inline: `style="--aa-load-cstagger: 0.04s; --aa-load-y: 0.8em"`.

## How the handoff works (no double animation)

The `instant` element is excluded from the FOUC guard, so the keyframe runs on the first frame — from `opacity: 0.01` (hidden but LCP-eligible) with `backwards` fill, so it reverts cleanly afterward without clobbering a later hover transform. The loader splits char-text the moment it parses (a `MutationObserver` wins the race even on a fast load). The keyframe isn't gated on init, so a fast `init()` can't cut it short; when the bundle lands the library **skips every `instant` element** (no tween, no re-split) and just flips `aa-ready`. So it works even if the bundle is slow or never arrives, and replays on the fresh element after a full-page or Barba navigation.

## When not to use it (use `load` instead)

`instant` is a first-paint optimisation for a *subset* of effects — it doesn't replace `load`. Reach for `load` (GSAP) when:

- You want a **cinematic / line-based** effect — masked line reveals, `text-slide-up`, `text-tilt`, `text-oval`, `text-rotate`. Those can't be instant (line wrapping depends on the final font, the very thing you'd be waiting for), so they stay on GSAP.
- The element is **below the fold or scroll-driven** — there's no first-paint wait to avoid, and you keep the full GSAP effect.
- You're on **Barba / a page-transition SPA**. After the first load the bundle is cached and `init()` runs inside the leave timeline, so there's no perceptible delay — and you usually want `load-once event:enter` so the reveal **syncs to the transition's peak**, which `instant` can't do (it fires immediately on the new container, ignoring the transition). `instant` still helps the *first cold load* there, but `event:enter` is the better hero trigger for the navigations.
- The LCP element is a **GSAP** entrance (not instant) — e.g. a hero image you're fading in via `load`. Then use [`aa-trigger="lcp"`](/animations/utilities/aa-trigger/). You **don't** need `lcp` *on top of* `instant` — the instant keyframe's `0.01` floor already makes the element an eligible LCP candidate, and the two can't be combined anyway (the library skips `instant` elements, so the `lcp` handling wouldn't run).

A typical page uses `instant` on the hero and `load` / `scroll` for everything else.
