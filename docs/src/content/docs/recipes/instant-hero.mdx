---
title: Instant hero entrances (Webflow)
description: Make above-the-fold hero animations paint on the first frame as pure CSS — before GSAP loads — with aa-trigger="instant". Element-level fade/slide/blur/zoom plus an optional no-GSAP splitter for per-character text cascades.
---

A normal `aa-trigger="load"` entrance is GSAP-driven, so its first frame can't paint until the GSAP bundle (and, for split text, the webfont) has downloaded and `init()` has run — often a second or two on a cold load. The hero sits hidden behind the FOUC guard the whole time.

`aa-trigger="instant"` removes that wait. The entrance is a **CSS keyframe** inlined in your `<head>`, so it paints on the **first frame the browser renders** — before GSAP exists. The keyframe runs to completion on its own, so whether the bundle arrives in 100 ms or 3 s makes no difference to the entrance. When it does arrive, the library detects `instant` and builds *no* GSAP tween for those elements — no re-animation, no flash.

| | `load` | `lcp` | `instant` |
| --- | --- | --- | --- |
| Engine | GSAP | GSAP (CSS 0.01 floor for LCP) | **CSS keyframe** |
| First frame | after bundle + init | LCP candidate early, fades after bundle | **animates immediately** |
| Split text | full SplitText | — | element-level, or char/word via the loader |
| Best for | below-the-fold, scroll | the one LCP image | above-the-fold hero copy |

## What you can animate

- **Element-level** (no extra script): `fade`, `fade-up` / `-down` / `-left` / `-right`, `zoom-in` / `zoom-out`, `blur`, `slide-up` / `-down` / `-left` / `-right`, `rotate*`. The motion comes from `aa-animate`; the eases match the GSAP named eases.
- **Per-character text** (with the loader, below): `text-fade`, `text-fade-up`/`-down`/`-left`/`-right`, `text-blur`, `text-scale` — split into characters and staggered, all in CSS.
- **Line-based text** (`text-slide-*`, `text-tilt-*`, `text-oval-*`, `text-rotate`) can **not** be instant — line breaks depend on layout and the final webfont, which aren't known before the bundle loads. Under `instant` they gracefully fall back to an element-level fade; use plain `load` (GSAP) if you need the real line effect.

```html
<h1 aa-animate="fade-up" aa-trigger="instant" aa-delay="0.1">
  Instant headline
</h1>
```

## 1. Head snippet (paste into Project Settings → Custom Code → Head Code)

This is self-contained: the keyframe, the eases it uses, and the split-unit layout rules are all inlined, because anything that only arrives in the non-blocking `dist` stylesheet isn't present on the first painted frame. Paste it **above** the alrdy-animate `<script>`/`<link>` tags.

```html
<!-- alrdy-animate · instant hero entrances (aa-trigger="instant") -->
<style>
  /* Eases needed at first paint (the dist :root copy arrives too late). */
  :root {
    --aa-ease-osmo: cubic-bezier(0.625, 0.05, 0, 1);
    --aa-ease-smooth: cubic-bezier(0.38, 0.005, 0.215, 1);
  }
  /* Split-unit layout — inline elements ignore transforms, so the char
     cascade needs inline-block from the very first frame. */
  .aa-word, .aa-char { display: inline-block; }
  .aa-sr-only { position: absolute; width: 1px; height: 1px; padding: 0;
    margin: -1px; overflow: hidden; clip: rect(0,0,0,0); white-space: nowrap; border: 0; }

  /* FOUC guard — instant (and lcp) opt out: they paint on purpose. */
  [aa-animate]:not([aa-ready]):not([aa-trigger~="lcp"]):not([aa-trigger~="instant"]) {
    visibility: hidden;
  }

  /* One parametric entrance keyframe, driven by per-direction custom props. */
  @keyframes aa-load-in {
    from {
      opacity: var(--aa-load-opacity, 0);
      translate: var(--aa-load-x, 0) var(--aa-load-y, 0);
      scale: var(--aa-load-scale, 1);
      rotate: var(--aa-load-rotate, 0deg);
      filter: blur(var(--aa-load-blur, 0));
    }
    to { opacity: 1; translate: 0 0; scale: 1; rotate: 0deg; filter: blur(0); }
  }

  /* Element-level entrance — whole element animates (unless the loader char-splits it).
     NOT gated on :not([aa-ready]): a fast init() must not cut the entrance short.
     `backwards` fill hides during any delay, then reverts to the natural state once
     the animation ends (so a later hover transform isn't clobbered). */
  [aa-trigger~="instant"][aa-animate]:not([aa-instant-split]) {
    animation: aa-load-in var(--aa-load-dur, 0.6s) var(--aa-ease-osmo) backwards;
    animation-delay: var(--aa-load-delay, 0s);
  }
  /* Per-character entrance — set up by the loader (adds aa-instant-split + the .aa-char spans). */
  [aa-trigger~="instant"][aa-instant-split] .aa-char {
    animation: aa-load-in var(--aa-load-dur, 0.6s) var(--aa-ease-osmo) backwards;
    animation-delay: calc((var(--char) - 1) * var(--aa-load-cstagger, 0.02s) + var(--aa-load-delay, 0s));
  }

  /* Element-level from-states (CSS only, no JS needed). */
  [aa-trigger~="instant"][aa-animate~="fade-up"]    { --aa-load-y:  2rem; }
  [aa-trigger~="instant"][aa-animate~="fade-down"]  { --aa-load-y: -2rem; }
  [aa-trigger~="instant"][aa-animate~="fade-left"]  { --aa-load-x:  2rem; }
  [aa-trigger~="instant"][aa-animate~="fade-right"] { --aa-load-x: -2rem; }
  [aa-trigger~="instant"][aa-animate~="zoom-in"]    { --aa-load-scale: 0.85; }
  [aa-trigger~="instant"][aa-animate~="zoom-out"]   { --aa-load-scale: 1.15; }
  [aa-trigger~="instant"][aa-animate~="blur"]       { --aa-load-blur: 20px; }
  [aa-trigger~="instant"][aa-animate~="slide-up"]    { --aa-load-opacity: 1; --aa-load-y: 100%; }
  [aa-trigger~="instant"][aa-animate~="slide-down"]  { --aa-load-opacity: 1; --aa-load-y: -100%; }
  [aa-trigger~="instant"][aa-animate~="slide-left"]  { --aa-load-opacity: 1; --aa-load-x: 100%; }
  [aa-trigger~="instant"][aa-animate~="slide-right"] { --aa-load-opacity: 1; --aa-load-x: -100%; }

  /* Char-split text from-states (chars inherit these from the element; opacity 0
     is the keyframe default, so every text entrance fades). Edit the distances /
     blur / scale here, and the per-char stagger via --aa-load-cstagger above. */
  [aa-trigger~="instant"][aa-animate^="text-"][aa-animate*="-up"]    { --aa-load-y:  0.4em; }
  [aa-trigger~="instant"][aa-animate^="text-"][aa-animate*="-down"]  { --aa-load-y: -0.4em; }
  [aa-trigger~="instant"][aa-animate^="text-"][aa-animate*="-left"]  { --aa-load-x:  0.4em; }
  [aa-trigger~="instant"][aa-animate^="text-"][aa-animate*="-right"] { --aa-load-x: -0.4em; }
  [aa-trigger~="instant"][aa-animate^="text-blur"]                   { --aa-load-blur: 8px; }
  [aa-trigger~="instant"][aa-animate^="text-scale"]                  { --aa-load-scale: 0; }

  /* Reduced motion: appear with no movement. */
  @media (prefers-reduced-motion: reduce) {
    [aa-trigger~="instant"][aa-animate],
    [aa-trigger~="instant"] .aa-char { animation-duration: 1ms !important; }
  }
</style>

<!-- Preload the hero webfont — required for char-split text, recommended always.
     Copy the real .woff2 URL from your Webflow font's @font-face. -->
<link rel="preload" as="font" type="font/woff2" crossorigin
      href="https://YOUR-SITE.website-files.com/…/HeroFont.woff2">

<!-- The loader: aa-delay staircase + per-character text splitting (no GSAP).
     Runs early and polls for the hero, so a plain (non-defer) tag is correct. -->
<script src="https://cdn.jsdelivr.net/npm/alrdy-animate@{{version}}/dist/loader.iife.js"></script>
```

That's the whole feature. Element-level `instant` works with just the `<style>` block — the loader is only needed for the delay staircase and per-character text.

## Combining with the slow-network fallback

If you also use the [slow-network + init-timeout fallback](/recipes/load-fallback/) (for `load` / `load-once` / `lcp` heroes), **merge the two into one inline `<head>` block — don't paste two separate ones.** They share the FOUC guard, and the only thing that matters is that there's exactly **one** guard line, excluding both `lcp` and `instant`:

```css
[aa-animate]:not([aa-ready]):not([aa-trigger~="lcp"]):not([aa-trigger~="instant"]) { visibility: hidden; }
```

(That's why the fallback recipe now ships this exact line — paste either order and the guards are identical.) Everything else just concatenates, because the two systems act on **disjoint** triggers:

- The fallback `<script>` timer targets `[aa-trigger~="load"]` / `[aa-trigger~="load-once"]` only — it never touches `instant`.
- The loader `<script>` + the instant `<style>` rules act on `instant` only.
- `instant` elements are self-revealing (the keyframe owns them), so they need no `aa-fallback` / `aa-timeout` safety — and the timeout's blanket `visibility: visible` is a harmless no-op for them (they're already visible).

So the combined block is: the fallback `<script>` **and** the loader `<script>`, plus one `<style>` with the single guard, the fallback keyframe/reveal rules, and the instant keyframe/rules/from-states.

:::tip[For the absolute-fastest char split]
A `<script src>` is still a network request. For a char-split hero that must animate on the very first frame, **inline** the loader instead: open `dist/loader.iife.js` (≈0.9 KB gzipped), copy its contents, and paste them inside an inline `<script>…</script>` in the head block above. No request, runs during head parse.
:::

## The `aa-delay` staircase

Add `aa-delay="<seconds>"` to stagger elements (headline, then subhead, then CTA). The loader reads it and sets the `--aa-load-delay` custom property the keyframe uses — so **any** delay value works, unlike a fixed list of CSS rules. `aa-duration` maps to `--aa-load-dur` the same way. Without the loader, set `style="--aa-load-delay: 0.1s"` by hand, or skip the staircase.

## Fonts

Making the hero instant *exposes* a font swap that the old slow reveal used to hide: with `font-display: swap` the text paints in the fallback font, then swaps to the webfont a beat later. For **char-split** text this also re-wraps the line when the font swaps. **Preload the hero webfont** (the `<link rel="preload">` above) so it's available on the first frame — this is required for `instant` on text and worth it everywhere.

## Customising the motion

Everything is in your inline `<style>`, so edit it directly — there's no dist copy to fight, and there are no motion values hidden in the loader JS (it only splits the text and sets `--char`; all the motion is CSS). To tune things:

- **Per-character stagger** — `--aa-load-cstagger` (default `0.02s`) in the `.aa-char` rule. Lower = tighter cascade.
- **Duration / ease** — the `animation` shorthand in the element + `.aa-char` rules (`--aa-load-dur`, the `var(--aa-ease-osmo)`).
- **How far / blurry each char starts** — the `[aa-animate^="text-"]…` from-state rules (`--aa-load-y`, `--aa-load-blur`, `--aa-load-scale`). These are inherited by the `.aa-char` spans, so editing them there changes the cascade.
- **Per-element override** — set any custom property inline, e.g. `style="--aa-load-cstagger: 0.04s; --aa-load-y: 0.8em"` on a single hero.

## How the handoff works (no double animation)

1. The element carries `aa-trigger="instant"` in the static HTML, so it's **excluded from the FOUC guard** and the CSS keyframe runs on the first frame (its `from { opacity: 0 }` + `animation-fill-mode: backwards` keep it hidden during any delay, then it reverts to its natural state once the animation ends — so no flash, and a later hover transform isn't clobbered).
2. The loader processes the hero the moment it parses (via a `MutationObserver`, so it wins the race even on a fast load): it sets the `aa-delay` custom property and, for char/word `text-*`, splits the text into `.aa-char` spans and adds `aa-instant-split`, switching the CSS from the element-level entrance to the per-character one.
3. The CSS keyframe is **not** gated on init, so it always runs to completion — a fast `init()` can't cut it short. When the GSAP bundle loads, the library's appear/text features and reduced-motion pass **skip every `instant` element** (no SplitText, no tween); `init()` still flips `aa-ready` for DOM consistency but it no longer affects these elements.

This means a `instant` hero works even if the bundle is slow or never arrives (it's self-revealing), and on a full-page or Barba navigation the keyframe simply replays on the fresh element.

## When not to use it

- For the single **largest** above-the-fold image, prefer [`aa-trigger="lcp"`](/animations/utilities/aa-trigger/) — it's tuned to count as the LCP candidate.
- For anything below the fold or scroll-driven, use plain `scroll` / `load` — there's no first-paint wait to avoid, and you keep the full GSAP effect (including masked line text).
