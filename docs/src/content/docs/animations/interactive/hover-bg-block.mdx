---
title: hover-bg-block
description: Direction-aware sliding background for buttons and links. Looks at the mouse-enter edge and slides the bg in from that direction.
---

import Demo from '../../../../components/Demo.astro'

Detects which edge the mouse crossed on enter/leave and slides a child `[aa-hover-bg]` element in from that direction. The element you put `aa-animate="hover-bg-block"` on is your interactive surface; place a positioned-absolute child with `aa-hover-bg` underneath your text content.

## Required structure

```html
<a class="my-btn" aa-animate="hover-bg-block">
  <span aa-hover-bg></span>
  <span class="label">Hover me</span>
</a>
```

## Attributes

| Attribute     | Default       | Notes                                   |
| ------------- | ------------- | --------------------------------------- |
| `aa-animate`  | (required)    | `hover-bg-block`                        |
| `aa-duration` | `0.5`         | seconds                                 |
| `aa-ease`     | `power2.out`  | any GSAP ease                           |

## Required GSAP plugins

`gsap` only — no plugins needed.

## Behavior

The hover effect is automatically disabled on devices without hover support (touch, e.g. mobile/tablet) via `(hover: hover)` matchMedia.
