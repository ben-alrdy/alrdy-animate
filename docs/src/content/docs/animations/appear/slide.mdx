`slide-up`, `slide-down`, `slide-left`, `slide-right` translate the element into place from the corresponding direction. **No opacity tween** вЂ” the element is visible the whole time. Pair with `overflow: hidden` on a parent if you want a clipping reveal effect.

For an entrance that *also* fades, use [fade-up/down/left/right](/animations/appear/fade/).

## Variants

| Variant       | Starts at                   |
| ------------- | --------------------------- |
| `slide-up`    | yPercent(100 Г— intensity)   |
| `slide-down`  | yPercent(-100 Г— intensity)  |
| `slide-left`  | xPercent(100 Г— intensity)   |
| `slide-right` | xPercent(-100 Г— intensity)  |

The offset is a percentage of the element's own size, so `aa-intensity="1"` slides exactly one element-width/height вЂ” perfect for fully clipping reveals when the parent has `overflow: hidden`.

## Attributes

| Attribute     | Default      | Notes                                                     |
| ------------- | ------------ | --------------------------------------------------------- |
| `aa-animate`  | (required)   | one of the variants above                                 |
| `aa-intensity` | `1`         | multiplier on the 100% (own-size) offset                  |
| `aa-duration` | (init)       | seconds вЂ” falls back to `init({ duration })` (default `0.6`) |
| `aa-delay`    | `0`          | seconds                                                   |
| `aa-ease`     | (init)       | any GSAP ease вЂ” falls back to `init({ ease })` (default `power4.out`) |
| `aa-stagger`  | (off)        | when set on a parent with children, staggers the children |
| `aa-scrub`    | (off)        | `true` or seconds вЂ” bind progress to scroll               |

## Required GSAP plugins

`gsap`, `ScrollTrigger`.

## Live demo вЂ” scrub-driven

Pair `slide-*` with `aa-scrub` to bind translation progress directly to scroll position. `aa-scrub="true"` snaps to scroll exactly; a numeric value (e.g. `"1"`) smooths it with that many seconds of lag.
