`slide-up`, `slide-down`, `slide-left`, `slide-right` translate the element into place from the corresponding direction. **No opacity tween** â€” the element is visible the whole time. Pair with `overflow: hidden` on a parent if you want a clipping reveal effect.

For an entrance that *also* fades, use [fade-up/down/left/right](/animations/appear/fade/).

## Variants

| `aa-animate` | Starts at                            |
| ------------ | ------------------------------------ |
| `slide-up`   | yPercent(100 Ă— distance)             |
| `slide-down` | yPercent(-100 Ă— distance)            |
| `slide-left` | xPercent(100 Ă— distance)             |
| `slide-right`| xPercent(-100 Ă— distance)            |

The offset is a percentage of the element's own size, so `aa-distance="1"` slides exactly one element-width/height â€” perfect for fully clipping reveals when the parent has `overflow: hidden`.

## Attributes

| Attribute     | Default      | Notes                                                     |
| ------------- | ------------ | --------------------------------------------------------- |
| `aa-animate`  | (required)   | one of the variants above                                 |
| `aa-distance` | `1`          | multiplier on the 100% (own-size) offset                  |
| `aa-duration` | `0.6`        | seconds                                                   |
| `aa-delay`    | `0`          | seconds                                                   |
| `aa-ease`     | `power4.out` | any GSAP ease                                             |
| `aa-stagger`  | (off)        | when set on a parent with children, staggers the children |
| `aa-scrub`    | (off)        | `true` or seconds â€” bind progress to scroll               |

## Required GSAP plugins

`gsap`, `ScrollTrigger`.
